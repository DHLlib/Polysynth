# Polysynth 功能特性

## 讨论模式
- **六顶思考帽**：六种思维角色（白帽·事实、红帽·情感、黑帽·批判、黄帽·乐观、绿帽·创意、蓝帽·主持人）围绕话题多轮协作
- **辩论赛**：正方四辩 vs 反方四辩，主持人控场，固定四轮（开篇立论、深化论点、攻辩驳论、总结陈词）
- 模式规则纯数据化（`config/modes/*.json`），可扩展新模式

## 实时流式交互
- WebSocket 实时传输 LLM 流式 token，浏览器内逐字显示各角色发言
- Banner 横幅提示轮次切换，TurnEnd 自动固化消息
- 连接状态指示器

## 参与者配置
- 每个角色独立配置：LLM 模型、名称、颜色、system prompt
- 颜色选择器支持 Hex ↔ ANSI 双向转换，CLI 与 Web 视觉统一
- 主持人全局统一管理（名称/模型/颜色跨模式共享，prompt 可按模式独立）

## 供应商管理
- 多 LLM 供应商动态配置（API Key、Base URL）
- 每个供应商可关联多个模型
- 模型与供应商绑定，运行时自动路由到正确的认证信息

## 轮次与话题
- 轮次数元数据驱动：`configurable` 标志控制是否可前台调整
- 六顶思考帽：1~10 轮可调；辩论赛：固定 4 轮

## 会话历史
- 侧边栏展示所有历史会话列表（话题、模式、状态、时间）
- 点击历史会话可回溯完整讨论记录
- 新建会话清空当前状态

## 双模式运行
- **Web 模式**：React + FastAPI + SQLite，浏览器实时观看
- **CLI 模式**：`python backend/main.py`，终端彩色输出，共享同一套核心引擎

## 数据持久化
- SQLite 异步存储：模式配置、参与者、会话记录、消息
- JSON 配置文件作为默认 seed，数据库作为运行时权威来源
- 运行时同时写入 `.jsonl` 文件备份
